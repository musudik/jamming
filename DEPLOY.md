# Deploying JamLyrics (Hybrid: Vercel + Render)

**Topology**

```
Browser ──> Vercel (React SPA)  ──API calls──>  Render (Express + Prisma) ──> Render Postgres
```

- **Frontend** → Vercel (static SPA)
- **Backend + Postgres** → Render (uses `backend/Dockerfile` + the `render.yaml` blueprint)

No code changes are required — both pieces are already containerized/configured.

---

## 0. Prerequisites

- Push this repo to GitHub (Vercel and Render both deploy from a Git repo).
- Accounts on [vercel.com](https://vercel.com) and [render.com](https://render.com) (both have free tiers).

---

## 1. Backend + database on Render

1. Render dashboard → **New → Blueprint** → select this repository.
2. Render reads [`render.yaml`](render.yaml) and proposes:
   - `jamlyrics-db` — a managed Postgres
   - `jamlyrics-backend` — a Docker web service built from `backend/Dockerfile`
3. Before **Apply**, set the two secrets marked `sync: false`:
   - `SEED_ADMIN_PASSWORD` — the admin password you want (used to log in).
   - `CORS_ORIGIN` — leave as a placeholder for now (e.g. `*`); you'll set the real Vercel URL in step 3.
4. **Apply.** Render builds the image, provisions Postgres, injects `DATABASE_URL`,
   and on boot the container runs `prisma migrate deploy` → seed → starts the API.
5. Note the service URL, e.g. `https://jamlyrics-backend.onrender.com`.
   Verify: open `https://jamlyrics-backend.onrender.com/api/health` → `{"ok":true}`.

> Free Render services sleep when idle and cold-start in ~30–60s on the next request.

---

## 2. Frontend on Vercel

1. Vercel dashboard → **Add New → Project** → import this repository.
2. **Root Directory → `frontend`** (important — the app lives in a subfolder).
   Vercel auto-detects Vite via [`frontend/vercel.json`](frontend/vercel.json).
3. Add an **Environment Variable**:
   - `VITE_API_BASE` = your Render URL from step 1 (e.g. `https://jamlyrics-backend.onrender.com`)
4. **Deploy.** Note the resulting URL, e.g. `https://jamlyrics.vercel.app`.

---

## 3. Connect the two (CORS)

1. Back in Render → `jamlyrics-backend` → **Environment** → set
   `CORS_ORIGIN` = your Vercel URL (e.g. `https://jamlyrics.vercel.app`) → save (it redeploys).
2. Open the Vercel URL and log in at `/admin/login` with `admin@jamlyrics.local` and the
   `SEED_ADMIN_PASSWORD` you set.

---

## 4. Load the songs into production

The production database starts empty (your local Telugu songs live only in the local Docker
volume). Load them into the deployed API:

```powershell
$env:API_BASE="https://jamlyrics-backend.onrender.com"
$env:ADMIN_EMAIL="admin@jamlyrics.local"
$env:ADMIN_PASSWORD="<the password you set on Render>"
node backend/scripts/load-songs.mjs
```

This creates the **Munich – Tollywood Jamming Night** event and imports all 30 songs
(ordered by number). Re-running it clears and reloads that event (idempotent).

---

## Notes & alternatives

- **Railway instead of Render:** create a project from the repo, set the service root to
  `backend/` (it auto-detects the Dockerfile), add a Railway Postgres plugin, and set the same
  env vars (`DATABASE_URL`, `JWT_SECRET`, `SEED_ADMIN_PASSWORD`, `CORS_ORIGIN`). Then point
  Vercel's `VITE_API_BASE` at the Railway URL.
- **Custom domain:** add it in Vercel, then update `CORS_ORIGIN` on the backend to match.
- **Security:** `JWT_SECRET` is auto-generated by Render; always set a strong
  `SEED_ADMIN_PASSWORD`. Rotate the seed admin or create a real admin for production use.
